# 灵语 - 语音识别输入项目

## Git 提交流程规范

### 前置要求
- 我是零 Git 基础用户，所有操作必须手把手指导：告诉我每一步点哪里、写什么、复制什么、粘贴到哪里
- 不要用专业术语，操作必须最简单

### 触发口令
当我说 **「把更新后的这个版本上传到代码库」** 时，自动启动全套上传流程。

### Commit 规范
- commit 描述绝不空白，必须和代码修改完全匹配
- 所有提交必须声明：`Claude AI 辅助开发，本人自主整合调试`
- commit 信息要简短规范，直接可复制使用

### PR 规范
- 单人仓库必须走：功能分支 → commit → 开 PR → 合并，保留完整记录
- **每个 PR 只做一件事**：一个 PR 只实现或修改单一功能；大功能必须拆成多个独立 PR 分步提交
- PR 描述必须完整合规，直接可复制使用
- 绝不生成空白描述
- PR 描述必须包含以下四项内容（我来撰写）：
  1. **标题**：一句话说明新增/修改了什么
  2. **功能描述**：说明该功能的作用与使用方式
  3. **实现思路**：简要说明技术选型或核心实现逻辑
  4. **测试方式**：如何验证该功能正常运行
- **合并后主分支必须可运行**：任何时候查看演示效果都能正常展示

### 版本管理
- 每次上传前先比对当前版本和上一仓库版本的代码差异
- 提炼修改的功能/代码内容
- 所有提交记录要分散，不一次性导入代码

- 禁止删除本文档
- 禁止让我写复杂命令
- 禁止让我理解 Git 原理
- 禁止暴露我不会操作的情况
