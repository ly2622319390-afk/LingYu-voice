# 灵语 - 语音识别输入项目

## Git 提交流程规范

### 前置要求
- 我是零 Git 基础用户，所有操作必须手把手指导：告诉我每一步点哪里、写什么、复制什么、粘贴到哪里
- 不要用专业术语，操作必须最简单

### 触发口令
当我说 **「把更新后的这个版本上传到代码库」** 时，自动启动全套上传流程。

### Commit 规范
- commit 描述绝不空白，必须和代码修改完全匹配
- 所有提交必须声明：`Claude AI 辅助开发，本人自主整合调试`
- commit 信息要简短规范，直接可复制使用

### PR 规范
- 单人仓库必须走：功能分支 → commit → 开 PR → 合并，保留完整记录
- PR 描述必须完整合规，直接可复制使用
- 绝不生成空白描述

### 版本管理
- 每次上传前先比对当前版本和上一仓库版本的代码差异
- 提炼修改的功能/代码内容
- 所有提交记录要分散，不一次性导入代码

### 禁止事项
- 禁止删除本文档
- 禁止让我写复杂命令
- 禁止让我理解 Git 原理
- 禁止暴露我不会操作的情况
